# Reconcista — сборка графа знаний Psycho

Ты — сильная модель. Архитектура проекта построена на разделении труда:

- **Бот (Qwen 14B, live)** только *захватывает*: ведёт диалог, пишет `raw/`,
  создаёт **черновые** концепты `status: draft` — БЕЗ связей и БЕЗ противоречий.
  Он намеренно не строит граф: он в этом слаб (плодит дубли и ложные конфликты).
- **Ты (вручную)** *строишь* выверенный граф из черновиков и сырья: промоушн
  `draft → stable`, дедуп/слияние, связи, реальные противоречия, переписывание
  `profile/`, осмысленный MOC, наглядный digest.

**Портрет носителя и настроение — НЕ твоя зона.** `personality/about.md`,
`personality/mood.md`, граф `mood/`, `user_prompt.md` ведёт отдельный скилл
**`depersonalization`**. Здесь — только знания (концепты/profile/digests).

## Что тебе можно менять, а что нельзя

**Можно (твоя зона — знания):**
- `concepts/<domain>/<slug>.md` — промоушн, слияние, связи, контр-callouts, статусы.
- `profile/<domain>.md` — переписывать из свалки цитат в связный обзор.
- `concepts/<domain>/<DOMAIN>.md` (MOC-нода, имя = тема заглавными) — пересобирать.
- `digests/<YYYY-WNN>.md` + `digests/_index.md` — создавать.
- `<base>/_tags.md` — реестр сквозных тегов.

**Нельзя (территория бота / другого скилла / источники истины):**
- `raw/**` — сырой лог, неприкосновенен (источник правды для evidence).
- `personality/**`, `mood/**`, `user_prompt.md` — зона скилла `depersonalization`.
- `_session.json`, `_state.json`, `.psycho/manifest.json`, `_index.md` — служебное бота.

**Безопасность:** вольт — git-репозиторий. **Перед любыми правками** сделай
checkpoint-коммит, **после** — финальный (см. Фазу 2). Бот не перезатрёт
выверенные (`stable`) концепты, потому что у них **русский `slug`**, а `safe_slug`
бота принимает только ASCII → запись отклоняется (см. «Наглядный граф»).
⚠️ Drift-detection на это НЕ полагается: manifest бота ключуется по пути, а
переименованный (русский) файл ему неизвестен → `check_drift` вернёт `False`.
Реальный барьер — русский slug, а не drift.

## Где вольт (multi-user)

Путь — из `VAULT_HOST_PATH` в `<project>/.env` (дефолт
`C:/Users/eva/YandexDisk/Obsidian/Psycho`). Данные пользователя — в
`<vault>/users/<user_id>/` (далее `<base>`). По умолчанию работай с **владельцем**
(`OWNER_TELEGRAM_ID`). **Никогда не смешивай данные разных пользователей.**
`.psycho/` и `.git/` — глобальные на корне (log — только чтение).

## Структура и форматы

```
<base>/
├─ raw/<YYYY-MM-DD>.md           # Q&A дня: «## Q<N> · HH:MM · <domain>», **Q:** / **A:**, ^Q<N>
├─ concepts/<domain>/<slug>.md   # узлы (кроме _*.md). Бот пишет draft, ты доводишь до stable
├─ profile/<domain>.md           # пока свалка цитат — переписываешь в обзор
├─ _tags.md                      # реестр сквозных тегов
└─ digests/                      # твои обзоры
```
Пути — относительно `<base>`. 10 доменов: `ethics, aesthetics, politics, everyday,
relationships, identity, mortality, nationality, knowledge, work`.

**Один ответ может затрагивать несколько тем — но домен концепта берётся из его
содержания** (как назначил бот в `observation.domain`), а не «всё в домен исходного
вопроса». При промоушене проверь, что домен концепта соответствует его сути; не
переноси силой в домен вопроса.

**Черновик бота** (вход): `slug: <kebab-ascii>`, `status: draft`, без связей, тело
часто в легаси-формате `## Подтверждения` / `## Открытые вопросы`. Это сырьё.

**Выверенный концепт** (что ты пишешь — `status: stable`):
```markdown
---
type: principle|value|preference|belief|claim
domain: <один из 10>
slug: <Русское имя файла>          # = имени файла (раздел «Наглядный граф»)
status: stable
supports: ["[[Русское имя]]"]
contradicts: ["[[Русское имя]]"]
derived_from: ["[[Русское имя]]"]
related: ["[[Русское имя]]"]
aliases: [старый ascii-slug, прежние формулировки]   # ascii — якорь дедупа
tags: [<DOMAIN-CAPS>, <сквозная-тема>, …]            # доменный тег = имя папки CAPS; из _tags.md
---
# Русское имя

> [!summary] TL;DR
> Законченное предложение в одну строку (его берёт MOC).

> [!quote]
> цитата из raw
> — из [[raw/<date>#^Q<N>]] · <дата время>

> [!question]
> открытый вопрос / probe
```
`tentative`/`contested` — твои промежуточные пометки, доводи их до `stable`.
Контр-противоречие — `> [!contradiction] vs [[Русское имя]]` в ОБА концепта +
симметричная связь `contradicts` во frontmatter обоих.

## Наглядный граф: русское имя = русский slug

Узлы графа Obsidian подписаны **по-русски** (принцип kepano: имя файла = заголовок,
ссылки ведут по имени, не по id). У `stable`-концепта на русском всё сразу: имя
файла, `slug`, все ссылки.
- Имя файла = `slug` = русское имя из `# Имя`: `Неприятие вялости в людях.md`.
- Все ссылки — `[[Русское имя]]` (frontmatter/тело/profile/MOC/digest).
- Старые ascii-slug → в `aliases` (якорь дедупа: бот предлагает ascii, `resolve_slug`
  находит по alias и не плодит дубль).
- **Бот не перезапишет stable** (русский slug ∉ ASCII → `save_concept`/`append_evidence`
  возвращают `None`). Новые цитаты копятся в `raw/` и подхватываются тобой.

**Санитайзинг имени (Windows-safe):** убери `< > : " / \ | ? *`, схлопни пробелы,
без точки/пробела в конце; не reserved-имя Windows; ≤~80 симв.; коллизия → ` 2`.
Переименование — только `git -C <vault> mv "<старое>.md" "<Русское имя>.md"`.

## Теги: сквозные темы + домен

Второе измерение графа: при «Show tags» каждый тег — узел, концепты с общим тегом
стягиваются. Польза — **сквозные мотивы через РАЗНЫЕ домены**. Теги — НЕ замена
`related` (смысловая связь = `[[ссылка]]`, тег — мотив-хаб).
- Теги в `tags:` frontmatter (без `#`), без пробелов (слова через дефис).
- **Доменный тег = имя папки ЗАГЛАВНЫМИ** (`aesthetics` → `AESTHETICS`) — узел-хаб
  домена, заменяет MOC-файл на графе. Ровно один, совпадает с `domain`.
- **Сквозные темы — русские** (`яркость`, `присутствие`…), 1–3 на концепт.
- **Закрытый реестр `<base>/_tags.md`**: новый тег появляется СНАЧАЛА там, потом в
  концептах. Лежит вне `concepts/` → в граф не попадает (источник правды для LINT).

**Граф-настройки (`.obsidian/graph.json`):** `showTags: true`, `showOrphans: false`,
`hideUnresolved: true`; MOC-файлы скрыты по имени (`-file:ETHICS …`); цветовые
группы по подпапкам `path:concepts/<домен>`, узлы-теги серые (нижняя группа).
Per-user сидится ботом из `bot/assets/graph.json` (существующий не трогает).
⚠️ Obsidian перезаписывает `graph.json` при правках в GUI — после внешних правок
Reload (Ctrl+R), настройки графа в GUI не меняй.

## Процедура: две фазы (proposal → apply)

### Фаза 0 — окно и сбор
1. ISO-неделя: `date +%G-W%V` → `YYYY-WNN`. Окно — последние 7 дней.
2. Прочитай: `raw/` за неделю; ВСЕ `concepts/*/*.md` (кроме `_*.md`) с frontmatter+телом,
   выдели `status: draft`; `profile/*.md`; `_tags.md`; `.psycho/log.md`.

### Фаза 1 — PROPOSAL (ничего не пишем)
Составь **план сборки**, покажи кратко в чате и запиши в
`<vault>/.psycho/reconcista-proposal-<YYYY-WNN>.md`. План:
1. **Промоушн черновиков.** Для каждого `draft`/`tentative`/`contested`: **merge** в
   существующий (указать в какой, перенести evidence+alias) / **promote** как новый
   `stable` (уточнить name/summary/type, выдать русский slug) / **drop** как шум.
   Проверь домен по сути концепта (не по теме вопроса).
2. **Связи** `supports / derived_from / related` (обоснование в строку).
3. **Реальные противоречия** — только настоящие. Если в `raw` человек явно отверг
   рамку — НЕ ставить, отметить снятым.
4. **Переписать `profile/<domain>.md`** — свалку цитат в обзор с кластерами и `[[ссылками]]`.
5. **MOC** — какие домены пересобрать.
6. **Новые цитаты к выверенным.** Пройди `raw/` недели: реплики, поддерживающие
   `stable`-концепт, добавь как `[!quote]` (бот этого не делает — барьер русского slug).
7. **Теги.** Каждому концепту доменный `<DOMAIN-CAPS>` + 1–3 сквозные из `_tags.md`.
   Новый мотив без тега — предложи тег в реестр с обоснованием.

Спроси: **«Применить план? да / нет / правки»**. Без «да» — стоп.

### Фаза 2 — APPLY (по подтверждению)
1. **Checkpoint:** `git -C <vault> add -A && git -C <vault> commit -m "reconcista <YYYY-WNN>: before apply" --allow-empty`.
2. Применяй план по порядку:
   - **merge:** перенести evidence/aliases в целевой, обновить summary, удалить лишний
     файл, переписать все `[[old]]` → `[[Русское имя survivor-а]]`; survivor переименовать
     в русское имя, старый ascii → в `aliases`.
   - **promote:** `status: stable`, причесать summary/name; имя файла = `slug` = русское:
     `git -C <vault> mv "<ascii>.md" "<Русское имя>.md"`, `slug: <Русское имя>`, ascii → `aliases`.
   - **связи:** симметрично во frontmatter обоих; цель = `[[Русское имя]]`.
   - **противоречия:** `contradicts` в обоих + `> [!contradiction] vs [[Русское имя]]`.
   - **evidence:** новые `[!quote]` из `raw/` к подходящим `stable`.
   - **теги:** `tags:` в stable; обнови `_tags.md`; один раз — граф-настройки.
   - **profile:** перезаписать `profile/<domain>.md` (обзор, `[[ссылки]]`).
   - **MOC:** пересобери ботом (НЕ `docker compose run` — поднимет ollama):
     ```bash
     # <repo> = корень проекта, <vault> = путь вольта, <user_id> = пользователь.
     MSYS_NO_PATHCONV=1 docker run --rm --entrypoint python \
       -v "<repo>":/app -v "<vault>":/vault -w /app --env-file <repo>/.env \
       psycho-bot:latest -c "from bot import userctx, moc; userctx.set_user(<user_id>); moc.rebuild_domain_moc('<domain>')"
     ```
     **MOC-ноду `<DOMAIN>.md` руками не пиши** — бот перестраивает MOC на каждом старте.
     Bot берёт summary как первую строку `[!summary]` (≤200 симв.) — она должна быть
     законченным предложением.
3. **Digest:** создай `digests/<YYYY-WNN>.md` по `digest-template.md`.
4. **Навигация:** дополни `digests/_index.md` (новый сверху).
5. **Финальный коммит:** `git -C <vault> add -A && git -C <vault> commit -m "reconcista <YYYY-WNN>: applied"`.

### Фаза 3 — LINT (проверка целостности, действие)
Механическое — чини, остальное — отчёт в чат:
- **Битые ссылки** `[[Русское имя]]` — почини после merge.
- **Инвариант русских имён** (stable): имя файла = `slug` = русское; есть ascii-alias.
- **Остаточные `status: draft`** среди обработанных — не должно быть.
- **Симметрия `contradicts`:** `[!contradiction]` содержит `[[wikilink]]` + парный `contradicts`.
- **`[!quote]` без атрибуции; `[!todo]` в `stable`** — почини (todo только в draft).
- **Осиротевшие** (0 связей, 1 evidence, давно `tentative`) — в отчёт, не трогай.
- **Теги:** каждый ∈ `_tags.md`; доменный = `domain` CAPS; без пробелов; 1 доменный + ≤3 сквозных.
- **Неприкосновенное не тронуто:** `raw/`, `personality/`, `mood/`, `_session.json`,
  `_state.json`, `.psycho/manifest.json`, `_index.md` (`git diff --name-only` checkpoint..HEAD).

## Закрытая схема callout-ов
Только эти типы (новые — расширять явно; валидируется в Фазе 3):

| Тип | Назначение | Обязательно |
|---|---|---|
| `[!summary]` | TL;DR концепта/недели | первая строка — законченное предложение ≤200 симв. |
| `[!note]` | нейтральное примечание, «снято пользователем» | — |
| `[!question]` | открытый вопрос / probe | — |
| `[!warning]` | напряжение позиций | суть столкновения |
| `[!contradiction]` | реальное противоречие | `[[wikilink]]` + парный `contradicts` |
| `[!quote]` | цитата из `raw/` | атрибуция `— из [[raw/<date>#^Q<N>]]` |
| `[!source]` | сессия/источник | ссылка на сессию |
| `[!todo]` | отложенное действие | **только** в `status: draft` |

Один callout = одна мысль, не вкладывать. `[!contradiction]` вместо тихого мержа.

## Разовая миграция существующих концептов
При первом прогоне после апгрейда (под checkpoint-коммитом): пройди все
`concepts/*/*.md`, для каждого выверяемого — переименуй в русское имя, `slug: <Русское имя>`,
старый ascii → `aliases`, перепиши все `[[old]]` → `[[Русское имя]]`. Легаси-тело
`## Подтверждения` приведи к callout-формату. Свежие `draft` не трогай — на промоушене.

## Digest
`digests/<YYYY-WNN>.md` — наглядный обзор для человека по `digest-template.md`
(callouts, `[[wikilinks]]`, frontmatter `type: digest`): что проступило, напряжения
(с пометкой «снято» где надо), что собрано/слито, вопросы-кандидаты на неделю.

## Если запускают «только обзор, без сборки»
Выполни Фазу 0 + сразу digest (read-only), пропусти Фазы 1–2. По умолчанию скилл
**собирает граф**.
