# Reconcista — граф мировоззрения 01-04

Ты — сильная модель. Live-бот только захватывает: задаёт вопросы по канонической
тройке `area/category/theme`, пишет `00_raw/` и создаёт draft-атомы. Он не должен
утверждать stable-связи и реальные противоречия на лету.

Твоя зона — выверенный граф мировоззрения:

- `01_Мироощущение/atoms/*.md` — эмоционально-чувственный уровень;
- `02_Миропонимание/atoms/*.md` — рационально-теоретический уровень без отдельного
  анализа знаний;
- `03_Ценностно-нормативная подсистема/atoms/*.md` — ценности, идеалы, нормы, табу;
- `04_Практический уровень/atoms/*.md` — воля, действия, стратегии, стиль жизни;
- `*/MOC.md` — оглавления областей.

## Нельзя трогать

- `00_raw/**` — источник истины, только читать.
- `05_Общее/about.md`, `05_Общее/profile.md`, `05_Общее/softskills.md`,
  `05_Общее/user_prompt.md`, `05_Общее/mask_frequencies*.json`.
- `01_Мироощущение/mood/**` кроме чтения как evidence эмоционального фона.
- `_session.json`, `_state.json`, `.psycho/manifest.json`, `_index.md`.

`05_Общее/summary.md` можно обновлять только в части кратких выводов по 01-04,
если задача явно просит собрать общий worldview-summary; иначе оставь depersonalization.

## Формат атома

```yaml
area: 02_Миропонимание
category: principles
theme: держать слово
type: principle
slug: Держать слово
status: stable
aliases: [derzhat-slovo]
evidence:
  - quote: "..."
    source: "[[00_raw/qna/2026-06-09#^Q12]]"
supports: []
contradicts: []
derived_from: []
related: []
influences: []
manifests_as: []
```

Связи всегда path-wikilink’ами:

`[[03_Ценностно-нормативная подсистема/atoms/Честность|Честность]]`

Инварианты:

- `contradicts` симметрична;
- `supports`, `derived_from`, `influences`, `manifests_as` направленные;
- self-link запрещён;
- битые ссылки запрещены;
- stable-атом имеет ясное русское имя, старый ascii-slug в `aliases`;
- один атом = одна мысль, не лонгрид.

## Типичные cross-layer связи

- `02_Миропонимание` supports/derived_from `03_Ценностно-нормативная подсистема`
  когда убеждение обосновывает ценность или норму.
- `03_Ценностно-нормативная подсистема` influences `04_Практический уровень`
  когда ценность влияет на действие/стратегию.
- `01_Мироощущение` manifests_as `04_Практический уровень`
  когда эмоциональный фон проявляется в поведении.
- `04_Практический уровень` contradicts `03_Ценностно-нормативная подсистема`
  когда поступки расходятся с заявленной ценностью.

## Процедура

1. Определи окно: обычно последние 7 дней или явно заданный период.
2. Прочитай `00_raw/qna`, `00_raw/notes`, `00_raw/sessions`, все atoms 01-04 и MOC.
3. Составь proposal:
   - какие draft/tentative атомы promote/merge/drop;
   - какие связи добавить внутри области;
   - какие связи добавить между областями;
   - какие реальные противоречия зафиксировать;
   - какие MOC пересобрать;
   - какие краткие выводы по 01-04 предложить для `05_Общее/summary.md`.
4. Спроси: `Применить план? да / нет / правки`. Без явного «да» не пиши.
5. Checkpoint: `git -C <vault> add -A && git -C <vault> commit -m "reconcista <period>: before apply" --allow-empty`.
6. Примени merge/promote/drop, связи, MOC, summary-фрагменты если разрешено.
7. Финальный коммит: `git -C <vault> add -A && git -C <vault> commit -m "reconcista <period>: applied"`.

## Lint

- Нет битых path-wikilink’ов.
- Нет self-link.
- `contradicts` симметрична.
- Нет дублей summary/name после merge.
- Нет orphan atoms без evidence; давние сироты вынеси в отчёт.
- MOC каждой затронутой области обновлён.
- `00_raw/**`, mood-анализ и файлы `05_Общее` вне согласованного summary не изменены.
