`05_Общее/summary.md` или отдельный недельный файл, если пользователь попросил.
Сохраняй порядок секций и стиль callout-ов. Пустые секции пропускай
(не пиши «нет данных» — просто убери заголовок).

## Правила

- **Wikilinks:** используй path-wikilink до атома:
  `[[02_Миропонимание/atoms/Русское имя|Русское имя]]`. Битая ссылка = ошибка,
  проверь имя файла в `<01-04 область>/atoms/`.
- **Callout-типы:** `summary`, `warning`, `note`, `todo`, `question` —
  стандартные Obsidian, рендерятся из коробки.
- **Связи:** явно называй тип связи: `supports`, `contradicts`, `derived_from`,
  `related`, `influences`, `manifests_as`.
- **Без воды.** Каждая строка несёт наблюдение. Не пересказывай ответы целиком —
  выделяй смысл.
- **Раздел «Здоровье базы» — это proposal, не действие.** Формулируй как
  рекомендации, не применяй. Механические фиксы (битые ссылки, инвариант
  русских имён, симметрия `contradicts`) выполняет отдельная **Фаза 3 — LINT**
  (см. SKILL.md); digest только показывает находки.
- **Если плагин Dataview есть** — можно добавить блок со свежими атомами:
  ```dataview
  TABLE area, category, theme, status FROM "01_Мироощущение/atoms" OR "02_Миропонимание/atoms" OR "03_Ценностно-нормативная подсистема/atoms" OR "04_Практический уровень/atoms" WHERE created >= date(<start>)
  ```
  Если не уверен, что Dataview установлен — обычный markdown-список (он
  отрендерится всегда).
